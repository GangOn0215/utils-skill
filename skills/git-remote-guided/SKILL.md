# Git Remote Guided

자동화 대신 **각 단계마다 사용자에게 확인**을 받으며 GitHub 연결을 진행합니다.
사용자가 흐름을 통제하고 싶거나 중요한 저장소를 다룰 때 사용합니다.

## 워크플로우

각 단계는 실행 **전에** 어떤 명령을 실행할지 보여주고 사용자 승인을 받은 뒤 진행합니다.

1. **상태 점검 보고** — 로컬 git 상태와 원격 상태를 요약해서 보여줌
   - 예: "현재 디렉토리는 git 저장소가 아닙니다. 원격에 커밋 N개 있습니다 (LICENSE, README.md 포함). 어떻게 진행할까요?"
2. **시나리오 제안** — `[[git-remote-auto]]`의 분기 시나리오(A~E)를 사용자에게 후보로 제시
3. **단계별 실행 확인** — 매 git 명령마다 실행 전 명령어와 결과 예상치를 보여줌
4. **커밋 메시지 검토** — 자동 생성한 메시지를 보여주고 수정 의사 확인
5. **푸시 직전 최종 확인** — 어떤 파일이 어떤 브랜치로 가는지 요약 후 사용자 OK 받고 푸시

## 반드시 확인 받는 시점

- 로컬 디렉토리 초기화 직전 (`git init`)
- 원격 추가 직전 (`git remote add`)
- 파일 스테이징 직전 (어떤 파일들이 포함되는지 목록 표시)
- 커밋 직전 (메시지 검토)
- 푸시 직전 (브랜치/원격 URL 재확인)
- 원격에 LICENSE/README 등이 있어 `fetch` 후 `reset --soft`가 필요할 때
- 기존 remote가 다른 URL일 때
- `.gitignore` 자동 생성/수정이 필요할 때

## 출력 형식 (단계마다)

```
📋 다음 단계: git init
명령어: git init && git branch -M main
영향: 현재 디렉토리에 .git 폴더 생성, 기본 브랜치를 main으로 설정
계속 진행할까요? (y/n)
```

마지막 단계 완료 시:

```
✅ 원격 연결 완료
저장소: https://github.com/user/repo
브랜치: main
커밋: abc1234
적용 시나리오: B (원격 파일 보존)
실행한 단계: 7개 / 사용자 승인: 7개
```

## 안전 규칙

- `[[git-remote-auto]]`의 안전 규칙을 모두 동일하게 준수
- 추가로, 자동 모드가 "조용히 처리"하는 부분도 **반드시 사용자에게 보고**:
  - `.gitignore`에 추가되는 패턴 목록
  - `reset --soft` 후 staged 상태 변화
  - 원격에서 가져온 파일 목록
- 사용자가 한 번 "전체 진행 OK"를 주면 그 이후 단계는 보고만 하고 일괄 실행해도 됨
  (단, D/E 시나리오 발생 시에는 다시 확인)

## 관련 스킬

- 한 번에 자동으로 처리하려면 `[[git-remote-auto]]` 사용
- 커밋 메시지 규칙은 `[[git-commit-helper]]` 와 공유
