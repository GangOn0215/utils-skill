## 📋 리뷰 항목 (우선순위 순)

### 1. 코딩 스타일 (PHP 5.6 + PSR-1/PSR-2)
- 들여쓰기: **4 spaces** (탭 금지)
- 줄 길이: 80자 soft / 120자 hard
- 클래스/메서드 열린 중괄호: **새 줄**
- 제어 구조 열린 중괄호: **같은 줄**
- 네이밍: 클래스 `PascalCase`, 메서드 `camelCase`, 상수 `UPPER_CASE`
- PHP 키워드(`true`, `false`, `null`) 모두 소문자
- 문자열: 싱글 쿼트 우선
- 배열: `array()` 문법 (PHP 5.6 호환)
- DocBlock(`/** */`) 필수
- 타입 힌트: 클래스/배열/콜러블만 (scalar는 PHP 7+)

### 2. 보안
- SQL 인젝션, XSS, CSRF
- 파일 업로드 취약점
- 개인정보 노출

### 3. 성능
- N+1 쿼리
- 불필요한 반복/연산
- 메모리 누수

### 4. 에러 처리
- try-catch 누락
- null/빈 값 체크
- 엣지 케이스 대응

### 5. 테스트 커버리지
- 단위/통합 테스트 누락 여부

### 6. 브레이킹 체인지
- 기존 코드 영향 범위, 마이그레이션 필요성

## 📊 출력 형식

````
✅ 잘한 점
- 인터페이스로 추상화가 깔끔하게 분리되어 있음
- DocBlock이 모든 public 메서드에 작성됨

⚠️ 개선 제안 (패치 포함)
- 23라인: $userId를 직접 SQL에 삽입 → 바인딩 사용 권장
  ```php
  // Before
  $sql = "SELECT * FROM users WHERE id = $userId";
  // After
  $stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
  $stmt->execute([':id' => $userId]);
  ```

❌ 치명적 문제 (반드시 수정)
- 41라인: 파일 업로드 시 확장자 검증 없음 → RCE 가능성

종합 점수: 6/10
````
